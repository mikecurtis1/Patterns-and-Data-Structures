XML & XSL

The following samples render various XML documents in the browser using XSL and CSS. The source code for each can be viewed through my GitHub democode repository.

  • Sonnet 29
    This is one of my favorite Shakespeare sonnets. And what better demonstration of a document type definition (DTD) than a sonnet? A sonnet isn't just a poem. A sonnet has a well defined structure with formal names for its parts: quatraine, couplet, rhyme scheme etc. I expressed these formal elements in my DTD (view source code) and in XML attributes.
  • A short list of films
    A film description offers an interesting example for a formal document definition. Not only can a film be described with rich metadata, but that data is often context-specific. For example: dates. A date associated with a film can describe both when the film was produced and also the date of the setting. Or consider awards. Awards can be give to the film - ex. National Film Registry - or to indivual cast and crew members - ex. best cinematography. In this example the definition includes many optional and implied parts and the XLT transformation helps handle the labeling and punctuation of this optional content.
  • A library catalog record
    This document is in MaRC XML format. MaRC is the standard metadata format for library catalog records and was developed by the Library of Congress. The MaRC metadata schema is very granular and identifies fields with alpha-numeric codes. This document is styled so that the data column has an approximately 50 character measure for readability.
  • Broome Community College Top 100 Books 2008
    The XML of this sample is the direct output of a report from Aleph - a popular library catalog system. Often these reports are tweaked and opened in MS Excel. But, in this case I created an XSL transformation to work directly on the unmodified report and prepare it for viewing. Ranking the books was accomplished in XSL, because it could not be accomplished in the Aleph report. Cover images come from the Google Books API based on ISBN's found in the report data.
  • Upstate Medical University METS record
    Metadata Encode and Transmission Standard (METS) is a standard maintained by the Library of Congress and is commonly used in digital libraries and archives. METS documents use their own namespace and encode some metadata with other schemata such as MODS and Dublin Core. This sample simply lists each datum with its element name and any relevant modifier attributes.
